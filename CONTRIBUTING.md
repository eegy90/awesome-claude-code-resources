## Resource Guidelines

Before adding a resource, please ensure it meets the following criteria:

### ‚úÖ Accepted Resources

1. **High-quality content** - Tutorials, documentation, tools, courses, or guides that provide genuine value
2. **Claude Code related** - Must be directly related to Claude Code, MCP, Agent SDK, or related Anthropic tools
3. **Accessible** - Free resources are preferred; paid content should be clearly marked
4. **Up-to-date** - Resources should be current and actively maintained when applicable
5. **Original source** - Link to the original source when possible

### ‚ùå Not Accepted

1. **Single blog posts** unless they are comprehensive guides or include unique insights
2. **Promotional content** without substantial educational value
3. **Duplicate resources** - Check if the resource already exists
4. **Broken links** - Ensure all links are working
5. **Twitter/X accounts, YouTube channels** (individual videos/playlists are fine)
6. **Content with licensing issues**
7. **Resources in languages other than English** (unless exceptionally valuable)

## Adding a Resource

### Quick Add (GitHub Web UI)

1. Go to the [README.MD](https://github.com/shahshrey/awesome-claude-code-resources/blob/main/README.MD)
2. Click the edit icon (pencil) at the top right
3. Find the appropriate category for your resource
4. Add your resource in the correct format (see below)
5. Create a Pull Request with a descriptive title

### Resource Format

**For links:**
```markdown
- :emoji: [Resource Title](https://example.com/resource-link)
```

**For repositories (in tables):**
```markdown
| :wrench: [owner/repo-name](https://github.com/owner/repo-name) | Stars count | Brief description |
```

### Formatting Rules

1. **Alphabetical order** - Add resources in alphabetical order within their category
2. **One resource per line** - Each resource should be on its own line
3. **Consistent emoji usage** - Use the appropriate emoji (see legend below)
4. **Brief descriptions** - Keep descriptions concise and informative
5. **No trailing slashes** - URLs should not end with `/`

## Pull Request Guidelines

1. **Descriptive title** - e.g., "Add: Claude Code Tutorial by Author Name"
2. **One category per PR** - If adding to multiple categories, consider separate PRs
3. **Fill out the PR template** - Provide all requested information
4. **Check for duplicates** - Search the README before submitting
5. **Test your links** - Ensure all links are working

### PR Title Format

- `Add: [Resource Name]` - For new resources
- `Update: [Resource Name]` - For updating existing resources
- `Fix: [Description]` - For fixing broken links or typos
- `Remove: [Resource Name]` - For removing outdated resources

## Contributing to the Website

The website is built with Jekyll and hosted on GitHub Pages.

### Prerequisites

- Ruby 2.7+
- Bundler
- Jekyll

### Local Development

1. Clone the repository:
```bash
git clone https://github.com/shahshrey/awesome-claude-code-resources.git
cd awesome-claude-code-resources
```

2. Install dependencies:
```bash
bundle install
```

3. Run the local server:
```bash
bundle exec jekyll serve --livereload
```

4. Open `http://localhost:4000` in your browser

### File Structure

- `css/` - Sass stylesheets
- `_includes/` - HTML partials (header, footer, head)
- `assets/js/` - JavaScript files
- `_config.yml` - Jekyll configuration

### Making Changes

- **CSS changes** - Edit files in `css/` folder
- **Layout changes** - Edit files in `_includes/` folder
- **Configuration** - Edit `_config.yml`
